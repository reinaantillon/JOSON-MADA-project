Cleaning

By inspecting the data as done above, we find some problems that need addressing:

First, there is an entry for height which says “sixty” instead of a number. Does that mean it should be a numeric 60? It somehow doesn’t make sense since the weight is 60kg, which can’t happen for a 60cm person (a baby). Since we don’t know how to fix this, we might decide to remove the person. This “sixty” entry also turned all Height entries into characters instead of numeric. That conversion to character also means that our summary function isn’t very meaningful. So let’s fix that first.

d1 <- rawdata %>% dplyr::filter( Height != "sixty" ) %>% 
                  dplyr::mutate(Height = as.numeric(Height))

hist(d1$Height)

Now we see that there is one person with a height of 6. That could be a typo, or someone mistakenly entered their height in feet. We assume that we know it was a mistaken entry as feet and convert to centimeters. Depending on your knowledge of the data, you might instead want to remove this person.

d2 <- d1 %>% dplyr::mutate( Height = replace(Height, Height=="6",round(6*30.48,0)) )

Height values seem ok now.

Now let’s look at the Weight variable. There is a person with weight of 7000, which is impossible, and one person with missing weight. To be able to analyze the data, we’ll remove those individuals as well.

d3 <- d2 %>%  dplyr::filter(Weight != 7000) %>% tidyr::drop_na()

Now checking the Gender variable. Gender should be a categorical/factor variable but is loaded as character. We can fix that with simple base R code to mix things up.

d3$Gender <- as.factor(d3$Gender)

Now we see that there is another NA, but it’s not NA from R, instead it was loaded as character and is now considered as a category. We’ll proceed here by removing that individual with that NA entry. Since this keeps an empty category for Gender, I’m also using droplevels() to get rid of it.

d4 <- d3 %>% dplyr::filter( !(Gender %in% c("NA","N")) ) %>% droplevels()

All done, data is clean now.

Let’s assign at the end to some final variable, this makes it easier to add further cleaning steps above.

#these are columns that should be numeric based on their names and data, but are currently classified as characters
num_cols <- c("plot_weight","total_seed_weight","calc_seed_weight_200",
              "seed_weight_200","seed_weight_100","seed_per_pound")

#convert "num stored as chr" columns into numeric
rawdata1 <- rawdata1 %>%
  mutate(across(all_of(num_cols), parse_number)) 

rawdata1 <- rawdata1 %>%
  # optional but helpful: trim whitespace so "   " becomes ""
  mutate(across(where(is.character), str_squish)) %>%
  # turn empty strings into NA
  mutate(across(where(is.character), ~ na_if(.x, ""))) %>%
  # turn "-9" strings into NA (only affects character columns)
  mutate(across(where(is.character), ~ na_if(.x, "-9"))) %>%
  # turn -9 numeric sentinel into NA (only affects numeric/integer columns)
  mutate(across(where(is.numeric), ~ na_if(.x, -9)))

Save data

Finally, we save the clean data as RDS file. I suggest you save your processed and cleaned data as RDS or RDA/Rdata files. This preserves coding like factors, characters, numeric, etc. If you save as CSV, that information would get lost. However, CSV is better for sharing with others since it’s plain text. If you do CSV, you might want to write down somewhere what each variable is.

See here for some suggestions on how to store your processed data: http://www.sthda.com/english/wiki/saving-data-into-r-data-format-rds-and-rdata

#save_data_location <- here::here("data","processed-data","processeddata.rds")
save_data_location1 <- here::here("data","processed-data","processeddata1.rds")
#saveRDS(processeddata, file = save_data_location)
saveRDS(processeddata1, file = save_data_location1)

Notes

Removing anyone who had “faulty” or missing data is one approach. It’s often not the best. based on your question and your analysis approach, you might want to do cleaning differently (e.g. keep individuals with some missing information).
